# 💻 Ghid GitHub pentru non-developeri: Portofoliu, proiecte și prezență online

GitHub nu este doar pentru programme. Este cea mai mare platformă de **cod și colaborare** din lume, dar și un loc excelent pentru a-ți construi **portofoliu**, a găzdui **site-uri web gratuite** și a demonstra **proiecte**. Acest ghid îți arată cum să folosești GitHub fără să știi să programezi.

---

## 🌟 De ce GitHub ca non-developer?

| Motiv | Beneficiu |
|-------|-----------|
| 🎨 Portofoliu profesional | Demonstrí competențe tehnice |
| 🌐 GitHub Pages | Site web gratuit și rapid |
| 📄 Markdown | Documentație simplă |
| 🤝 Colaborare | Contribute la proiecte open source |
| 🔍 CV vizibil | Angajatorii caută GitHub |

## 📝 Partea 1: Concepte de bază explicate simplu

### Git vs GitHub

| Termen | Ce este |
|--------|---------|
| **Git** | Programul care ține evidența modificărilor |
| **GitHub** | Website-ul care găzduiește proiectele tale |
| **Repository (repo)** | Un folder de proiect |
| **Commit** | O „salvare" cu istoric |
| **Push** | Trimite modificările online |
| **Pull** | Preia modificările de online |

### Analogia simplă

```
Git = Google Docs versiuni
GitHub = Google Drive pentru cod
Commit = Version history în Docs
```

## 👤 Partea 2: Crearea contului și profilului

### Setup profil profesional

1. Creează cont pe [github.com](https://github.com)
2. Alege un **username profesional** (preferabil numele tău)
3. Adaugă **fotografie de profil**
4. Completează **bio** (50-100 cuvinte despre tine)
5. Adaugă **location** și **website**

### Profil complet = Profil profesional

```
✅ Username: nume-prenume
✅ Bio: Product Designer cu 5 ani experiență
✅ Location: București, România
✅ Website: portofoliul tău
✅ Avatar: poză profesională
```

## 📁 Partea 3: Repository — Crearea unui portofoliu

### Crează un repo pentru portofoliu

1. Click pe **„+"** → **„New repository"**
2. Nume: `portofoliu` sau `cv`
3. Description: „Portofoliul meu de proiecte"
4. **Public** (ca să fie vizibil)
5. ✅ **Add a README file**
6. Click **Create repository**

### Structura unui README de portofoliu

```markdown
# 👋 Salut! Sunt [Nume]

## Despre mine
[2-3 propoziții despre tine]

## 🎯 Competențe
- HTML & CSS
- Design UX/UI
- Figma
- Canva

## 📂 Proiecte

### [Proiect 1](link)
Descriere scurtă • [Live Demo](link) • [Figma](link)

### [Proiect 2](link)
Descriere scurtă • [Live Demo](link) • [Behance](link)

## 📫 Contact
- [LinkedIn](link)
- [Email](email)
- [Website](link)

## 🏆 Achievements
- [Certificare](link)
- [Premiu](link)
```

## 🌐 Partea 4: GitHub Pages — Site web gratuit

### Activează GitHub Pages

1. În repo-ul tău, mergi la **Settings**
2. Scroll la **Pages** (în sidebar)
3. Source: **Deploy from a branch**
4. Branch: **main** / **(root)**
5. Click **Save**

În câteva minute, site-ul tău va fi live la:
`https://username.github.io/repo-name`

### creează site cu GitHub Pages

#### Opțiunea 1: HTML simplu

1. Creează un repo nou
2. Adaugă fișiere HTML, CSS
3. Activează GitHub Pages
4. Site-ul e live!

#### Opțiunea 2: Șabloane

Folosește șabloane gratuite:
- [HTML5 UP](https://html5up.net) — design-uri responsive
- [Start Bootstrap](https://startbootstrap.com) — șabloane variate
- [Templated](https://templated.co) — minimalist

## 🤝 Partea 6: Contribuie la Open Source

### De ce să contribui?

- Înveți de la code profesionist
- Demonstrí competențe
- Conectezi cu comunitatea
- Îmbunătățești proiecte reale

### Cum să începi

1. Găsește un proiect cu **good-first-issue**
2. Fork (copie proprie)
3. Fă modificări
4. Submit Pull Request
5. Așteaptă review și merge

### Proiecte welcoming pentru beginners

- Documentație (corrigă greșeli)
- Traduceri (română → engleză)
- UI/UX improvements
- Bug reports bine scrise
